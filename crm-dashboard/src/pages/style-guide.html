공통 레이아웃 시스템

1920×1080 기준의 시각 비율을 유지하면서 1280, 1440, 1600, 1920, 2560+ 업무용 해상도에서 Header, LNB, card gap, content padding이 자연스럽게 스케일되도록 정의한 canonical layout guide입니다.

Preview

실제 AppShell과 같은 토큰으로 Header/LNB/3-column dashboard의 상대 비율을 보여줍니다.

Responsive Shell
Header · var(--layout-header-height)

Spec

고정 px 대신 clamp 기반 토큰을 사용합니다. 아이콘 24px 기준은 유지하고, 컨테이너와 여백만 스케일됩니다.

Header

기준값은 64px. 작은 업무용 해상도에서는 56px까지 축소되어 dense dashboard 느낌을 유지합니다.

  • heightclamp(56px, 3.5vw, 64px)
  • padding-xvar(--layout-header-pad-x)
  • gapvar(--layout-header-gap)

LNB

기준값은 96px. 작은 해상도에서는 72px까지 줄어들며 placeholder icon은 24px 기준을 유지합니다.

  • widthclamp(72px, 5vw, 96px)
  • buttonclamp(36px, 2.08vw, 40px)
  • icon24px fixed

Dashboard Density

카드 비율은 유지하고 gap/padding만 조절해 정보량과 여백의 균형을 맞춥니다.

  • content gapclamp(10px, .72vw, 14px)
  • card gapclamp(8px, .63vw, 12px)
  • card paddingclamp(14px, .94vw, 18px)

Responsive Rules

모바일 전환이 아니라 고해상도 업무용 대시보드의 시각 밀도 보정 규칙입니다.

≤1280Header/LNB 최소값 근접, gap 10px, card gap 8px.
1440compact desktop. scale 0.94, 과한 여백 방지.
1600standard desktop. scale 0.97, 1920에 가까운 밀도.
1920canonical baseline. Header 64px, LNB 96px.
2560+Header/LNB는 상한 유지, page padding/gap만 소폭 증가.

Usage

신규 페이지와 컴포넌트는 아래 토큰을 사용하고, LNB/Header에 고정 px를 직접 쓰지 않습니다.

Do

Shell, 페이지 padding, 카드 간격은 layout token을 사용합니다.

  • shellvar(--layout-header-height)
  • lnbvar(--layout-lnb-width)
  • pagevar(--layout-shell-pad-x)

Don’t

Header 64px, LNB 96px, dashboard gap을 개별 페이지에서 직접 선언하지 않습니다.

  • 금지height: 64px
  • 금지width: 96px
  • 금지gap: random px

Principle

구조를 바꾸지 않고 밀도만 보정합니다. 카드 비율과 정보 우선순위는 화면 크기와 무관하게 유지합니다.

  • keep3-column ratio
  • scalespacing only
  • capshell max size
